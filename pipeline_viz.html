PreToolUse Security Guard

🔫 Fires before any Bash tool invocation
Scans the bash command string for dangerous patterns. If matched, exits with code 1 to block the command entirely.
Blocked patterns: rm -rf · DROP TABLE · sudo rm
python3 -c "
import sys, json
d = json.load(sys.stdin)
cmd = d.get('tool_input',{}).get('command','')
blocked = ['rm -rf','DROP TABLE','sudo rm']
if any(x in cmd for x in blocked):
    sys.exit(1)  # ← BLOCKS the tool call
else:
    print('ok')
"
Why it helps: CLAUDE.md forbids rm -rf anywhere. This hook enforces that rule automatically without relying on the LLM to remember.

PostToolUse Write Audit Log

📝 Fires after every Write tool call
Appends a timestamped entry to src/data/hook_log.txt each time any agent writes a file.
echo "[hook] File written at $(date)" \
  >> src/data/hook_log.txt
Why it helps: Creates a passive audit trail across all 5 pipeline stages. 18 entries were logged during the last full run — one per JSON file written.

Stop Pipeline Completion Check

✅ Fires when the entire Claude run completes
Checks whether src/outputs/aggregated.json exists. Prints success or warning accordingly.
python3 -c "
import os
if os.path.exists('src/outputs/aggregated.json'):
    print('[Stop hook] Run complete.
          Outputs in src/outputs/')
else:
    print('[Stop hook] Warning:
          aggregated.json not found')
"
Why it helps: Acts as a final gate. If the aggregator failed silently, this hook surfaces the failure immediately instead of letting broken output flow into xlsx/pptx/pdf skills.

SubagentStop Subagent Completion Log

🤖 Fires when any subagent finishes execution
Reads the subagent type from stdin and echoes it to stdout, providing a live breadcrumb trail of which agents have completed.
python3 -c "
import sys, json
d = json.load(sys.stdin)
agent = d.get('agent_type','unknown')
print(f'[hook] Subagent done: {agent}')
" 2>/dev/null || true
Why it helps: Since the 5 subagents run sequentially, this hook provides real-time progress visibility. The || true ensures a bad stdin never crashes the pipeline.

⚙️ Settings & Permissions (.claude/settings.local.json)

Network access is explicitly scoped to Yahoo Finance only:

{
  "permissions": {
    "allow": [
      "WebFetch(domain:query1.finance.yahoo.com)"
    ]
  }
}
This means the fetcher agent cannot accidentally call any other external URL — a minimal-privilege network policy.
